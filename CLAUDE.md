# slashuntitled.com — guide for Claude

Portfolio site for slash-untitled. Live at <https://slashuntitled.com>. Static site generated by Nuxt 3 + Tailwind, auto-deployed to GitHub Pages on every push to `main`.

## How to edit content

**All editable text** lives in [`content/site.json`](./content/site.json) — display name, project list, archive list, About bio, blog link, social handle, contact email. This is the single source of truth. Never hard-code strings into Vue components.

**Project images** live in `public/projects/<slug>/`. Drop `.jpg/.jpeg/.png/.webp/.gif/.avif` files in. The build lists them automatically and renders them on the matching project page. The folder name **must exactly match** the `folder` field in `site.json` (lowercase, hyphens, no spaces).

## Common tasks

### Add a new project

1. Create `public/projects/my-new-project/` and drop photos in
2. Add to `selectedWork` in `content/site.json`:
   ```json
   { "title": "My New Project", "slug": "my-new-project", "folder": "my-new-project" }
   ```
3. `slug` and `folder` should match — `slug` becomes the URL (`/my-new-project`), `folder` is where the images live

No Vue file edits needed. The `pages/[slug].vue` template handles all project pages.

### Move a project from active to archive

In `site.json`, move the entry from the `selectedWork` array to the `archive` array. Same shape.

### Update the About text or any social/contact info

Edit `content/site.json`. That's it.

### Change the design

CSS lives in [`assets/css/main.css`](./assets/css/main.css) — typography, font registration, peach inner-glow frame. Layout shell is [`app.vue`](./app.vue). Page-specific layouts are in `pages/`. The site deliberately mirrors ellenole.com's brutalist minimalism — small black text on white, single column, no decoration except the glow frame. Run design changes by Mark/slash-untitled before pushing — the look is the brand.

## Local dev

Requires Node 22+ (see `.nvmrc`).

```bash
npm install
npm run dev          # http://localhost:3000, hot reloads
npm run generate     # produces .output/public/ — same thing CI does
```

## Deployment

Push to `main` → [`.github/workflows/deploy.yml`](./.github/workflows/deploy.yml) runs `npm run generate` and publishes to <https://slashuntitled.com>. Takes ~1 minute. Watch live runs at <https://github.com/MarkClubine/slashuntitled.com/actions>.

For small content edits (text, new project, photo drops) — push directly to `main`.
For redesigns or anything that could break the build — feature branch + PR.

## Conventions

- **Slugs:** lowercase, hyphens only. No spaces, no caps, no underscores. They become URLs.
- **Folder names** under `public/projects/` match slugs exactly.
- **Font:** `assets/fonts/Arabic-font-2013.ttf` is the default for all text, registered in `main.css`. Don't add other fonts without coordinating — typography is part of the brand.
- **Image filenames inside project folders** can be anything; they're listed alphabetically.

## Password gate (current)

The site is gated by a client-side password prompt while it's in development.

- **Default password:** `slashuntitled`
- **Change it:** edit `app.config.ts` → set `gate.passwordHash` to the SHA-256 hex of your new password. There's a snippet in the file showing how to compute it from bash or a browser console.
- **Disable when launching:** set `gate.enabled = false` in `app.config.ts`, delete the `Disallow: /` line in `public/robots.txt` (or delete the file), and push.
- **Localhost is auto-bypassed** so dev isn't annoying.
- **Soft, not hard.** The page HTML, photos, and the password hash all download to the browser. It blocks casual visitors and search crawlers, not a determined attacker. If real protection is needed, move to a Cloudflare Worker proxy in front of Pages.

## Don't

- Commit `node_modules/`, `.output/`, `.nuxt/` — gitignored, keep them that way.
- Hard-code copy into Vue components — edit `content/site.json` instead.
- Use folder names with spaces or capitals — we migrated away from those.
- `git push --force` or `git commit --amend` on `main` — both Mark and slash-untitled have admin and force-pushing breaks the other's local clone.
- Add Vercel / Netlify / other deploy configs — we're on GitHub Pages, single source.
- Add `@nuxt/image`, ESLint, or other tooling without a real reason. Site is small; tooling is overhead.

## Background — why things are the way they are

- **Why Nuxt for a static site?** Mark wanted to match ellenole.com's stack (also Nuxt). Nuxt's `nuxi generate` produces fully static HTML; the deployed site has no server. The Nuxt machinery is build-time only.
- **Why GitHub Pages?** Free, owned-by-GitHub, auto-cert. The CI workflow exists because Vue/TS/Tailwind need a build step — *something* has to compile. Choosing Pages over Vercel was Mark's call.
- **Why is the image-listing API in `server/api/`?** It runs only during prerender (build time), reading `public/projects/<folder>/`. The deployed `.output/public/` is fully static; no server route ships.
- **Why `Arabic-font-2013.ttf` for all text?** slash-untitled's choice — supports Latin glyphs and gives the site its distinctive typography.
- **Why is the accessibility button in the bottom-right not interactive?** It's a visual reference to ellenole.com. If we want it to actually do something, that's a separate task.

## Identity / git config

The repo is at `MarkClubine/slashuntitled.com` (Mark's personal GitHub, not his work account). Both Mark and slash-untitled are admins.

If your GitHub email is set to private (default for new accounts), pushes will fail with `GH007: Your push would publish a private email address`. Fix by configuring git to use your GitHub-provided noreply email:

```bash
# Find your noreply email at github.com/settings/emails — looks like:
# 123456789+yourusername@users.noreply.github.com
git config user.email "ID+username@users.noreply.github.com"
git config user.name "Your Name"
```
